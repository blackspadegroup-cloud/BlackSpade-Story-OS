# BSOS Markdown 标准

> 本文档定义 BSOS Vault 中所有 Markdown 文件的格式规范。

---

## 1. 标题层级

### 1.1 规则

| 层级 | Markdown | 用途 | 每文件使用次数 |
|------|----------|------|--------------|
| H1 | `# 标题` | 文档标题（必须与 frontmatter title 一致） | 仅 1 次 |
| H2 | `## 标题` | 主要章节 | 不限 |
| H3 | `### 标题` | 子章节 | 不限 |
| H4 | `#### 标题` | 小节 | 不限 |
| H5 | `##### 标题` | 细节分类 | 谨慎使用 |
| H6 | `###### 标题` | 极少使用 | 尽量避免 |

### 1.2 硬性规则

- **H1 必须与 frontmatter 的 `title` 字段一致**
- H1 后跟一个空行，再接文档简介或目录
- 标题前后各空一行（文件开头的 H1 除外）
- 标题后不可直接跟另一个标题（须有正文内容分隔）
- 标题中不可使用 Obsidian 链接（`[[link]]`）

### 1.3 示例

```markdown
---
title: "BSOS 创作宪法"
---

# BSOS 创作宪法

> 副标题/引言

## 第一条：揭示优于解释

内容...

### 子要点

更具体的内容...
```

## 2. 正文格式

### 2.1 段落

- 段落之间空一行
- 段落内不空行
- 每行建议不超过 80 个中文字符（避免过宽）

### 2.2 强调

| 样式 | Markdown | 用途 |
|------|----------|------|
| 粗体 | `**文本**` | 关键术语、重要概念 |
| 斜体 | `*文本*` | 轻微强调、外来语 |
| 粗斜体 | `***文本***` | 极少使用 |
| 删除线 | `~~文本~~` | 废弃内容、修改标记 |

### 2.3 列表

**无序列表：**
```markdown
- 项目一
- 项目二
  - 嵌套项目
  - 嵌套项目
- 项目三
```

**有序列表：**
```markdown
1. 第一步
2. 第二步
   1. 子步骤
   2. 子步骤
3. 第三步
```

**任务列表：**
```markdown
- [x] 已完成
- [ ] 进行中
- [ ] 待办
```

### 2.4 引用

```markdown
> 第一层引用
>
> > 第二层嵌套引用
```

引用用于：
- 宪法原文引用
- 角色语录
- 外部来源标注

### 2.5 代码块

**行内代码：** `` `code` `` — 用于文件名、编码、标签

**代码块：**
````markdown
```yaml
title: "示例"
type: kernel
```
````

语言标识必须填写（yaml, json, markdown, python 等）。

### 2.6 分隔线

```
---
```

- 三个或以上 `-`
- 前后各空一行
- 一章内最多使用一次

## 3. Obsidian 内部链接

### 3.1 基本链接格式

| 语法 | 说明 | 示例 |
|------|------|------|
| `[[文件名]]` | 链接到文件（去除扩展名 .md） | `[[000_CONSTITUTION]]` |
| `[[文件名|显示文本]]` | 带别名链接 | `[[000_CONSTITUTION|创作宪法]]` |
| `[[文件名#标题]]` | 链接到指定标题 | `[[000_CONSTITUTION#第一条]]` |
| `[[文件名#^块ID]]` | 链接到块引用 | `[[001_PRINCIPLES#^principle-1]]` |

### 3.2 链接规则

- **路径引用：** 如果文件在子文件夹中，使用相对路径：
  - `[[STANDARDS/000_NAMING_CONVENTION]]` ✅
  - `[[00_KERNEL/STANDARDS/000_NAMING_CONVENTION]]` ✅

- **编码优先：** 对于编号文件，优先用编码引用：
  - `[[C-001]]` 优于 `[[C-001_首章标题]]`

- **链接文本：** 链接到外部时，使用 `|` 别名以获得可读文本：
  - `[[000_CONSTITUTION|宪法]]` ✅
  - `[[000_CONSTITUTION]]`（在正文中突兀）⚠️

### 3.3 断裂链接检测

- 所有 `[[链接]]` 必须指向存在的文件
- 文件重命名后，必须更新所有指向它的链接
- 建议使用 Obsidian 的重命名功能（自动更新链接）

## 4. 嵌入（Embed）

```markdown
![[文件名]]
![[文件名#标题]]
![[文件名#^块ID]]
```

嵌入用于：
- 在索引文件中显示完整子文件内容
- 在剧情弧线中嵌入角色 DNA 摘要
- 在章节中嵌入场景卡

---

## 5. 标签（Tags）

### 5.1 行内标签

```markdown
正文中的 #kernel 标签
```

- 标签前必须有空格或行首
- 标签由字母、数字、连字符组成，无空格
- 示例：`#canon` `#needs-review` `#wip`

### 5.2 标签位置

标签应集中出现在：
1. **YAML frontmatter** 的 `tags` 字段中
2. 正文中**仅在需要标记特定段落**时使用行内标签

---

## 6. 表格

### 6.1 格式规范

```markdown
| 左对齐 | 居中 | 右对齐 |
|:-------|:----:|-------:|
| 内容   | 内容  | 内容   |
| 内容   | 内容  | 内容   |
```

### 6.2 表格规则

- 表头必须有
- 对齐标记（`:---`）必须有
- 表头和内容之间不能有空行
- 表格前后各空一行
- 建议表格不超过 5 列
- 单元格内避免换行

---

## 7. 脚注

```markdown
正文内容[^1]

[^1]: 脚注内容写在这里。
```

- 脚注编号从 1 开始递增
- 脚注定义放在文件末尾
- 建议谨慎使用脚注，能用正文说明的尽量不用脚注
